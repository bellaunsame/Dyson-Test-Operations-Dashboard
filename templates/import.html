Excel Requirements Guide

To ensure the Test Ops Dashboard can load and display your project timelines, your Excel file must follow the standard schema. Below are the required and optional columns:

1. File & Sheet Structure

  • Each tab/sheet in the workbook represents an active project (e.g. 873, 893).
  • The sheet named Master Data (case insensitive) is ignored by the system.

2. Required Columns (Critical)

The system maps your columns using fuzzy naming rules. Ensure your headers match or are similar to the following:

System Field Accepted Column Headers (Fuzzy Matches) Description
Category category, test category, group, device, product Groups test methods together on the dashboard.
Test Method method, test method, method name, name Name of the test (e.g., Drop Test).
Test Number number, test number, test #, ref, code Unique test method identifier (e.g., TM-003440).

3. Timeline & Phase Columns (Required for Gantt Chart)

The Gantt chart charts test progress through four chained phases: Proto, DVT, EVT, and PVT. Each phase requires three parameters:

  • Qty (Sample Quantity): Number of samples (e.g. Proto1, DVT1).
  • Weeks (Duration Weeks): Number of weeks (e.g. Proto1_Date (Week), DVT1_Date (Week)2).
  • Days (Duration Days): Remainder days (e.g. Proto1_Date (Day), DVT1_Date (Day)3).

4. Recommended Metadata

  • Start Date: Customizable starting date for the test method (e.g., Start Date, YYYY-MM-DD). If omitted, start date defaults to today.
  • Defect Qty: Number of failed samples (e.g. defect qty, defects).
  • Comments: Test notes or failure reasons (e.g. comments, notes).
